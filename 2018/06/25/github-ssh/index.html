ssh-keygen 生成SSH KEY

ssh 公钥认证是ssh认证的方式之一。通过公钥认证可实现ssh免密码登陆，git的ssh方式也是通过公钥进行认证的。
在用户主目录下有一个.ssh的目录，ssh-keygen 可用来生成ssh公钥认证所需的公钥和私钥文件

Mac OS

  1. 当前用户目录，.ssh目录下生成
  • ssh-keygen
    • -t 创建秘钥类型，
    • -f 指定文件名称，不指定默认为id_ras,id_ras.pub两根文件，如github多个账号或者一台系统登录多个环境：github，gitlab，可生成多个秘钥。
cd ~/.ssh
ssh-keygen -t rsa -f id_ras_github  -C "your_email@example.com"

输出如下：过程中提示输入密码，如果如不需要则直接回车，

Generating public/private rsa key pair.
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
Your identification has been saved in test.
Your public key has been saved in test.pub.
The key fingerprint is:
SHA256:4I4KncfJ2rExleoCQUzTn6bBk5yUto6zZL7oOK5OxoQ your_email@example.com
The key's randomart image is:

  1. 多账号配置，在.ssh目录下创建config文件，配置多账号环境：
# github your_email1@example.com
Host github_1
        HostName github.com
        PreferredAuthentications publickey
        IdentityFile ~/.ssh/id_rsa_github_1

#github your_email2@example.com
Host github_2
        HostName github.com
        PreferredAuthentications publickey
        IdentityFile ~/.ssh/id_rsa_github_2

Github 配置

  1. 复制公钥到github

  2. 本地git配置取消全局user、email配置,项目中设置 origian

#取消全局配置
git config --global --unset user.name
git config --global --unset user.email

# 查看项目 origin 
git remote -v

#重新设置项目 origin
git remote remove origin
git remote add origin git@github1:your_email1/project_name
